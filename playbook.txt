■■ OVERVIEW ■■

What This App Does
A web app that stops users losing money to hidden subscription renewals. The landing page teaches what
the app does before asking for login — then guides the user through a complete demo with realistic
sample data.

Core User Journey

1

Land on page

Understand the problem and solution — no login required

2

Click Try Demo

See all subscriptions instantly with sample data

3

Click Cancel

See protection options and cancellation methods

4

Get Alert

When a charge happens, generate dispute proof instantly

Design Principles
■ Modern, premium, polished — high-value fintech vibes, not a student project.
■ Every screen is self-explanatory. No tooltips or help text needed.
■ Responsive: works beautifully on phone, tablet, and desktop.

■■ PRANJAL'S ROLE ■■

pranjal — Backend + Data + 4 UI Screens
pranjal

Backend + Data + 4 UI Screens

4
screens

First 2 Hours on Day 1
1. Finalize database schema: users, subscriptions, renewals, cancellations, blocks, disputes
2. Lock API contracts — define all endpoints and their return shapes
3. Create GitHub board with issues assigned by day and owner

What You're Building
■ Build backend API (Node.js + Fastify) that serves all app data
■ Implement subscription tracking, renewal prediction, and cancellation lifecycle
■ Own user authentication and session management
■ Build 4 UI screens: Auth, Dashboard, Subscription Detail, Renewal Calendar
■ Ensure demo mode works without any external credentials
■ Own API QA: test all endpoints with Postman, validate demo data, catch any 500 errors

Definition of Done
✓ Auth flow is secure, fast, and clean UX
✓ Dashboard API returns monthly spend, next renewal date, and risk level
✓ Subscription Detail shows full info, history, and action buttons
✓ Renewal Calendar displays all charge dates color-coded by risk
✓ All APIs respond under 500ms, no errors in demo data
✓ CI passes and Railway deploy succeeds automatically
✓ QA sign-off: all API endpoints return correct data, full demo flow error-free
